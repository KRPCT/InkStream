# 创作模式

为长篇写作设计的模式：把作品拆成「章节-场景」结构来组织，配合角色设定库、段落聚焦、字数目标与场景概要，帮助你专注地把一部书写完。

## 开启创作模式

模式切换入口在窗口底部状态栏左侧的模式指示器（一个圆点加模式名）。

1. 点击状态栏的模式指示器。
2. 在弹出菜单中选择「Creative · 长篇创作」。

切到创作模式后，左侧边栏会出现章节-场景导航树，编辑器顶部会显示场景概要卡，右侧面板换成「Codex」和「场景概要」两个标签页。

## 章节-场景导航树

切到创作模式后，左侧边栏文件树上方会多出一块「章节」导航区，把工作区按「文件夹即一章、文件夹里的文件即一个场景」来组织。

- 工作区里的每个顶层文件夹是一「章」，章名就是文件夹名。
- 文件夹内的每个 Markdown 文件（`.md` / `.markdown` / `.txt`）是一个「场景」。
- 直接放在工作区根目录、没归进任何文件夹的文件，会被收进一个名为「未分章」的分组。
- 章和场景都按自然顺序排列（数字会按大小排，所以「第2章」排在「第10章」前面）。
- 以点号开头的隐藏目录（如 `.git`）不会出现在导航树里。

每个场景行左侧有一个状态色点，中间是场景名，右侧是该场景的字数。

- 场景名优先取文件 frontmatter 里的 `title`，没有则用去掉扩展名的文件名。
- 字数只统计正文，frontmatter 元数据不计入。
- 点击任意场景即可在编辑器中打开它，当前打开的场景会高亮显示。
- 当前正在编辑的场景，字数会随你的输入实时变化；其他场景的字数在工作区结构变化时刷新。

如果工作区里没有可识别的章节或场景，「章节」导航区不会显示。

## 场景状态色点

每个场景的写作状态来自该文件 frontmatter 里的 `status` 字段，对应一个颜色圆点：

- `status: draft` —— 草稿
- `status: revised` —— 已修
- `status: final` —— 定稿

没有写 `status` 字段、或写了无法识别的值，都按「草稿」处理。把鼠标停在场景行上，提示气泡会显示「场景名 · 状态 · 字数」。

## Codex 角色 / 地点 / 设定卡

Codex 是你作品的设定库：角色、地点、世界观设定各占一个文件，写到正文里时会被自动认出来并标记。

### 建立 Codex 条目

1. 在工作区根目录下建一个名为 `Codex` 的文件夹。
2. 在 `Codex/` 里为每个条目建一个 Markdown 文件。
3. 在文件 frontmatter 中至少写两项：`type` 和 `name`。

frontmatter 字段约定：

- `type:` —— 条目类型，取 `character`（角色）、`location`（地点）或 `lore`（设定）三者之一。其他值的条目会被忽略。
- `name:` —— 条目的规范名称，也是正文里被识别的触发词。必填，缺失则该条目不生效。
- `aliases:` —— 别名，单行书写、用英文逗号分隔（例如 `aliases: 阿深, 深哥`）。每个别名也会成为触发词。
- `summary:` —— 悬停卡上显示的简介。不写时自动取正文的第一段作为简介。

只有同时具备有效 `type` 和 `name` 的文件才会被当作 Codex 条目。

### Codex 面板

右侧面板切到「Codex」标签页，可以看到全部条目列表，每行显示条目名和类型标签（角色 / 地点 / 设定）。

- 点击任意条目即可打开对应的设定文件。
- 编辑完设定文件后，点面板右上角的刷新按钮重新扫描 `Codex/`，正文里的提及高亮会随之更新。
- 切换 / 打开工作区时也会自动重新扫描。

如果还没有任何条目，面板会提示你在 `Codex/` 文件夹放入条目。

### 提及高亮与悬停卡

当正文里出现某个条目的名称或别名时，这段文字会被标上点状下划线。把鼠标停上去，会弹出一张卡片，显示条目名、类型，以及简介。

- 同时存在多个条目时，较长的触发词优先匹配。
- 拉丁字母 / 数字按词边界识别（例如条目「John」不会命中「Johnson」里的片段）；中文等无词间空格的文字按子串匹配。
- 高亮只是视觉标记，不改变正文内容，也不影响光标、选区和中文输入。

## Focus Mode 段落聚焦

按 **F11** 开启或关闭 Focus Mode。开启后，光标所在的段落保持正常显示，其余段落被淡化，帮助你专注于当前正在写的这一段。

- 段落以空行为界划分。光标移动到哪一段，哪一段就高亮，其余淡化。
- 这是一个全局开关，对所有打开的文档都生效，再次按 F11 即可关闭。
- 在应用内 F11 用于切换 Focus Mode，不会触发系统全屏。

## 今日字数目标进度

创作模式下，状态栏可以显示你今天的写作进度条。

### 设置目标

1. 打开设置，进入「编辑器」一节。
2. 在「今日字数目标」里填入目标字数（填 0 表示关闭该进度显示）。

### 进度显示

设定目标且当前为创作模式时，状态栏右侧会出现一支笔图标加「已写 / 目标」数字和一条进度条。

- 进度统计的是「今日净写入」：累计你今天编辑产生的字数增量，跨午夜会自动归零重新计算。
- 打开一篇已有文档不会把它的存量字数算进今天，只有此后的编辑才计入。
- 切换不同场景继续写，进度会跨文档累计。
- 达到目标后，进度条颜色变为「定稿」色以示完成。

把鼠标停在进度区域上，会显示「今日已写 X / 目标 Y 字」。

## 场景概要卡

为每个场景写一句话概要，方便在不打开正文的情况下回顾这一场写了什么。

### 写概要

在场景文件 frontmatter 中写 `summary:` 字段（单行），例如：

```
---
title: 雨夜的相遇
status: draft
summary: 林深在码头初遇陌生人，约定三日后再见。
---
```

### 查看概要

- 创作模式下，打开有 `summary` 的场景时，编辑器顶部会出现一张可折叠的「场景概要」卡，点标题行可以展开 / 收起。没有概要的场景不显示这张卡。
- 右侧面板的「场景概要」标签页也会显示当前场景的概要内容。

概要卡是只读展示。要修改概要，直接编辑场景 frontmatter 里的 `summary` 行即可，卡片会随之更新。

## frontmatter 元数据约定

创作模式的多项功能都靠场景与 Codex 文件开头的 frontmatter 驱动。frontmatter 写在文档最前面，用一对 `---` 包起来，每行一个字段。

场景文件常用字段：

- `title:` —— 在导航树里显示的场景名（不写则用文件名）。
- `status:` —— 写作状态：`draft` / `revised` / `final`（不写按草稿）。
- `summary:` —— 场景概要（单行）。

Codex 文件字段：

- `type:` —— `character` / `location` / `lore`（必填）。
- `name:` —— 条目名称与触发词（必填）。
- `aliases:` —— 别名，单行逗号分隔。
- `summary:` —— 悬停卡简介（不写则取正文首段）。

注意事项：

- frontmatter 各字段都写成单行（`字段: 值`），值里可以含空格（如英文标题）。
- frontmatter 由你手写，写错或留空不会报错，只是对应功能回退到默认（如 `status` 缺失按草稿、`summary` 缺失不显示概要卡）。
