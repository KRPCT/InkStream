# 双向链接与知识网络

用 `[[ ]]` 在笔记之间建立链接。支持自动补全和点击跳转，链接目标不存在时会顺手建好；反链面板和全库索引则用来查看哪些笔记引用了当前文件。

## 什么是 wiki-link

wiki-link 是用双方括号包住目标的链接写法，让一篇笔记指向另一篇。它可以写在正文任意位置，渲染时显示为带下划线的链接样式。

语法为：

```
[[目标#标题^块|别名]]
```

其中只有目标是必填的，后面三段都可选，可任意组合或省略：

- `[[目标]]`：最简形式，链接到名为「目标」的笔记。
- `|别名`：链接显示成别名，但实际指向「目标」。例如 `[[2024-总结|去年回顾]]` 在正文中显示为「去年回顾」。
- `#标题`：指向目标笔记中的某个标题段落，例如 `[[方法论#实验设计]]`。
- `^块`：指向目标笔记中的某个文本块，例如 `[[读书笔记^p3]]`。

注意事项：

- wiki-link 必须写在同一行内，从 `[[` 到 `]]` 之间不能换行，否则不会被识别为链接。
- 空的 `[[]]` 不构成链接。
- 链接里的 `[[`、`]]`、`|` 这些结构符号在渲染时会自动隐藏，只显示别名（若有）或目标文本。

## 输入 `[[` 自动补全

输入 `[[` 后会自动弹出当前工作区的文件清单，边打字边模糊筛选，支持中文文件名。

- 列表按文件名相关度排序，单次最多展示 50 个候选。
- 候选项的副标题会显示该文件的相对路径，便于区分同名文件。
- 选中某项后会插入 `[[文件名]]`（自动去掉 `.md` 后缀，光标落在 `]]` 之后）。

## 点击跳转与「不存在即建」

按住 `Ctrl`（macOS 上为 `Cmd`）点击一个 wiki-link 即可跳转：

1. 先在工作区内按精确相对路径查找目标，找不到再按文件名匹配（裸名 `[[笔记名]]` 即可命中，多个同名时取第一个）。
2. 命中目标时，在当前编辑器内直接打开它。
3. 目标笔记不存在时，会自动新建 `目标.md`、打开它、刷新文件树，并弹出提示「『目标.md』不存在，已新建并打开。」

补充说明：

- 不按修饰键的普通点击不会跳转，而是把光标放到该处、显示链接的源码以便编辑。
- 同一套 `Ctrl`/`Cmd` 点击手势也适用于普通 Markdown 链接：`http(s)` 外链会用系统默认浏览器打开，工作区内的相对路径链接会在编辑器中打开；指向工作区之外或无法解析的链接不会有动作。

## 反链面板（含未链接提及）

反链面板显示「谁引用了当前这篇笔记」，是看清知识网络入向关系的地方。

- 打开位置：右侧面板的「反链」标签页（标准模式下右侧面板包含 大纲 / 反链 / 局部图谱 三个标签）。
- 内容随当前打开的文档自动更新，分为两组：
  - **反向链接**：用 `[[ ]]` 明确链接到当前文件的其他笔记。
  - **未链接提及**：正文里提到了当前文件名、但还没有建立 `[[ ]]` 链接的笔记。可据此把零散提及补成正式链接。
- 每组标题后带数量统计；点击任意一行即可在编辑器中打开对应笔记。
- 当前文件没有任何反链与提及时，面板显示「暂无反向链接」的空态提示。

注意事项：

- 「未链接提及」依赖全文索引的三元组匹配，文件名少于 3 个字时不会进行提及检索。
- 反链结果会排除文件自身，「未链接提及」也会自动剔除已经建立链接的笔记，避免重复。

## 全库索引与重建

反链、未链接提及等功能依赖一份覆盖工作区全部 `.md` 笔记的全文索引（保存在工作区内的 `.inkstream/index.db`）。

- **自动重建**：每次打开工作区时，索引会全量重建一次，使其与磁盘上的笔记保持一致。
- **增量更新**：会话过程中，笔记自动保存写盘后会更新对应条目的索引，不需要手动操作。
- 索引重建在后台进行，不会阻塞文件夹的打开与编辑。

## Ctrl+P 快速打开文件

按 `Ctrl+P` 打开「快速打开」面板，可在整个工作区里凭文件名快速跳到任意笔记。

- 在输入框中键入关键字，按文件名模糊筛选，支持中文。
- 结果行标题为文件名，副标题为相对路径，便于区分不同目录下的同名文件。
- 选中某项后会在编辑器中打开它。
