# 文件导出：HTML / PDF / DOCX

把当前正在编辑的 Markdown 文档导出成可分享的 HTML、PDF 或 Word（DOCX）。所有转换都在本地完成，不依赖任何外部服务或命令行工具。

## 怎么导出

标题栏 **文件菜单 → 导出为 →** 选 HTML / PDF / DOCX 其一；也可在命令面板（`Ctrl+Shift+P`）执行「文件：导出为 HTML / PDF / DOCX」。导出的是**当前文档**。

- **HTML / DOCX**：弹出系统保存对话框，选好位置即落盘。
- **PDF**：弹出系统打印对话框，在目标里选「**另存为 PDF**」保存（也可直接打印到纸）。

## 三种格式

- **HTML**：单文件、自包含——内联了一套适合阅读与打印的排版样式，双击就能在任何浏览器里打开，不依赖外部资源。
- **PDF**：用系统打印通道生成，版式与 HTML 一致，适合定稿分发与打印。
- **DOCX**：标准 Word 文档，标题、段落、加粗 / 斜体、列表、引用、代码、表格都会映射成对应的 Word 元素，可继续在 Word / WPS 里编辑。

## 内容如何转换

- Markdown 的标题、强调、列表、引用、代码块、表格、链接、图片都会正确转换。
- **数学公式**：`$...$`、`$$...$$`、` ```math ` 经 KaTeX 渲染为公式；`latex` / `typst` 围栏块在导出中以代码块形式保留源码。
- **Wiki 双向链接** `[[...]]`：导出为其显示文字（有别名取别名）。
- 安全：导出产物会在浏览器里打开，正文中的脚本、`javascript:` / `data:` 等危险链接在导出时一律被中和，不会被执行。

## 品牌标识

导出的文件会带上「**Made with InkStream**」标识：

- **页脚**：默认在文末附一行「Made with InkStream」。不想要可在 设置 → **通用 → 导出附带 InkStream 标识** 关闭。
- **元数据**：无论页脚开关如何，生成器标识（含版本号）始终写入文件元数据（HTML 的 `<meta generator>` / DOCX 的核心属性）。
