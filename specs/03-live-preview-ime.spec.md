> **本清单是 EDIT-06 永久回归门。** 它是全项目最高风险件（`view.composing` 冻结纪律）的护栏：
> 一旦冻结回退即吞字 / 候选窗跳位 / 上屏错乱。
>
> **本阶段及 Phase 4 / 5 / 8 / 9 的所有装饰类变更，合并前必须逐项勾验通过。** 任何触及
> CodeMirror 装饰层（块级 StateField / 行内 ViewPlugin / atomicRanges）的改动都要重跑本清单。
>
> 与 `specs/01-workbench.spec.md` 同构：入库公开可审计、人工逐项勾验。在力所能及范围内已补

> 自动桩无法复现真实候选窗行为，不可替代真机逐项勾验。

## 0. 范围与平台门

- **Windows 为本阶段硬门**：下方 4 输入法 × 4 场景类矩阵在 Windows 真机逐项勾验通过，方可合并装饰类变更。
- **macOS（注音 / 拼音）、Linux（fcitx / ibus）留 Phase 12 打包阶段补测**——本阶段不阻断，届时以同结构矩阵回归。
- 执行时机：Live Preview 链接 / 装饰落地后（Wave 2 之后）逐项执行；jsdom 无法复现真实候选窗行为，故全部为真机手测。

## 1. 通用判据（每格勾验四项，全部满足才算通过）

每个「输入法 × 场景」单元格在输入中文时，须同时满足以下四项，缺一即判失败：

1. **候选窗不跳位**：组合期（`compositionstart`→`compositionend`）候选窗位置稳定，不随装饰重算闪移。
2. **上屏无错乱**：`compositionend` 上屏的文字顺序、内容与所选候选完全一致，无错位 / 重复 / 顺序颠倒。
3. **无吞字**：组合期与上屏后均无字符丢失（含连续快速输入、组合中途删改）。
4. **刷新无字号 / 行高跳动**：`compositionend` 后装饰刷新瞬间，所在行字号、行高、缩进零位移
   （承接 UI-SPEC「显标记、保排版」不跳契约；中文正文行高不低于 1.5）。

## 2. 输入法覆盖（D-11，4 项）

| 代号 | 输入法 | 选取理由 |
|------|--------|----------|
| IME-A | 微软拼音 | 系统内置基准，最常见路径 |
| IME-B | 搜狗拼音 | 市占第一，第三方候选窗行为差异大 |
| IME-C | 双拼模式 | 码长 / 上屏时序与全拼不同 |
| IME-D | 手写 / 语音面板 | 非标准一次性上屏路径，绕常规组合事件序列 |

## 3. 场景 / 装饰位置覆盖（D-12，4 类）

| 代号 | 场景类 | 具体落点 |
|------|--------|----------|
| S-1 | 行首块标记处 | `##` / `-` / `>` / `1.` 后紧跟输入中文 |
| S-2 | 行内标记内 | `**` 加粗内 / `` ` `` 行内代码内 / `[]()` 链接文本内 |
| S-3 | 空文档 / 空行首字 | 无前文上下文的合成；含列表空项、表格空单元格 |
| S-4 | 多行块内续写 | 表格单元格 / 引用块 / 列表项续行的连续输入 |

## 6. 失败处置

任一单元格四判据有一项不满足，即判本次装饰变更不可合并：

1. 定位回退点：多为组合期未短路装饰重建（`view.composing` / `isComposing` 漏判）或 `compositionend` 刷新破坏盒模型。
2. 修复后**重跑整张矩阵**（不可只补失败格——冻结纪律是全局不变量）。
3. 修复与重验结论记入对应阶段 SUMMARY 的偏差 / 验证小节。
